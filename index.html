Kurzanleitung für Javascript Session Storage von Marius

Wozu braucht es diese WEB-API

Session Storage ist ein Teil der Web Storage API, die es Webanwendungen ermöglicht, Daten direkt im Browser des Benutzers zu speichern.
Im Gegensatz zu Cookies, die vom Server gesendet und empfangen werden, erfolgt der Zugriff auf den Session Storage und dessen Manipulation ausschließlich über JavaScript auf der Client-Seite.
Hier eine einfache Erklärung, was Session Storage ist und wie er funktioniert:

Was macht das Session Storage aus?

Speicherort im Browser: Session Storage ermöglicht es, Daten temporär im Webbrowser des Benutzers zu speichern.
Lebensdauer: Die gespeicherten Daten bleiben erhalten, solange die Browser-Tab-Session aktiv ist. Sie werden gelöscht, sobald der Tab oder der Browser geschlossen wird.
Kapazität: Bietet mehr Speicherplatz als Cookies (ca. 5MB oder mehr je nach Browser).
Pro Tab isoliert: Jeder Tab hat seinen eigenen isolierten Speicherbereich, sodass Sessions verschiedener Tabs einander nicht beeinflussen.

Unterscheidung zum Cookie

Lebensdauer: Das Cookie bleibt erhalten, auch nach dem schließen eines Tabs oder dem Browser. Hier von ausgenommen: Das Session-Cookie.
Kapazität: Das Session Storage verfügt über ein Speicherplatz von bis zu 5mb. Der Cookie hingegen nur über 4KB.
Zugriff Das Cookie kann über verschiedene Tabs hinweg aggieren. Das Session Storage nur innerhalb des aktuelle Tabs.
Serverkommunikation Das Cookie kommunziert per Server-Client-Anfrage. Das Session-Storage ist hingegen ein rein Clientseitig.

Grundlegende Operationen

Speichern von Daten
Um Daten im Session Storage zu speichern, verwendet man die setItem(key, value) Methode. Der key ist dabei der Bezeichner für den zu speichernden Wert.
sessionStorage.setItem('meinSchluessel', 'meinWert');
Abrufen von Daten
Daten werden mit der getItem(key) Methode abgerufen. Gibt den Wert zurück, der dem angegebenen Schlüssel zugeordnet ist.
let meinWert = sessionStorage.getItem('meinSchluessel'); console.log(meinWert); // Gibt 'meinWert' aus
Löschen von Daten
Einzelne Daten können mit removeItem(key) entfernt werden, während clear() alle Daten im Session Storage des Benutzers für die aktuelle Domain löscht.
sessionStorage.removeItem('meinSchluessel'); // Löscht den spezifischen Eintrag sessionStorage.clear(); // Löscht alle Einträge im Session Storage

Verwendungsszenarien

Formulardaten speichern: Zwischenspeichern von Benutzereingaben, falls die Seite neu geladen wird.
Benutzerpräferenzen: Speichern von vorübergehenden Einstellungen wie Spracheinstellungen oder Themenauswahl.
Zwischenspeichern von Daten für Single Page Applications (SPAs): Temporäres Speichern von Daten, während der Benutzer die Anwendung nutzt.

    const container = document.querySelector("#container");
    const text = document.querySelector("#text");
    
    const totalTime = 7500;
    const breatheTime = (totalTime / 5) * 2;
    const holdTime = totalTime / 5;
    
    breathAnimation();
    
    function breathAnimation() {
        text.innerHTML = "Einatmen!";
        container.className = "container grow"; // Growanimation in css
    
        setTimeout(() => {
        text.innerHTML = "Halten";
    
        setTimeout(() => {
            text.innerHTML = "Ausatmen!";
            container.className = "container shrink"; // shrinkanimation in css
        }, holdTime); // nach dem Halten ausatmen
        }, breatheTime); // Nach einem Atemzyklus halten
    }
    
    setInterval(breathAnimation, totalTime);
